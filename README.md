# 心愈 AI 健康诊断建议算法

本仓库放置“心愈”App 中诊疗方案模块的核心算法代码。

它承接上一步四维诊断结果：

- 心理状态
- 身体问题
- 八字/能量叙事判断
- MBTI 判断

然后生成：

- AI 状态总结
- 健康管理结论
- 能量图数据
- 个性化疗愈建议
- 7 日健康计划
- 适合推荐给用户的心愈广场模块

## 文件说明

```text
src/xinyu-prompts.ts
```

AI 大模型使用的系统 Prompt 和用户 Prompt 模板，负责控制输出边界、语气和 JSON 结构。

```text
src/xinyu-health-diagnosis-skill.ts
```

AI 健康诊断建议 Skill。输入四维诊断结果，输出用户当前属于哪类场景：

- 健康稳定：状态很好，不制造焦虑，输出阳光鼓励和维持建议
- 问题修复：存在压力、睡眠、身体或情绪失衡，输出调理优先级和疗愈建议
- 高风险：出现自伤想法、严重失眠、严重抑郁等信号，优先输出安全提示和专业求助建议

```text
src/xinyu-weekly-plan-skill.ts
```

AI 健康诊断周度计划 Skill。根据诊断结果生成 7 日健康计划，每天包含主题、目标、任务、推荐模块和打卡问题。

```text
examples/xinyu-skill-example.ts
```

三种测试输入样例：

- 健康场景
- 问题场景
- 高风险场景

这些样例不是 App 页面内容，只是给研发测试算法输出使用。

## 产品边界

心愈不是纯医疗诊断或治疗 App，代码中统一使用：

- 状态
- 倾向
- 风险提示
- 健康管理建议
- 调理建议
- 可尝试

避免使用：

- 确诊
- 治疗
- 治愈
- 保证疗效

八字判断只用于能量叙事、节律观察和生活方式建议，不作为医学判断依据。
